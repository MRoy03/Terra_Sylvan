Terra Sylvan
Developer Guide

A complete technical reference for every layer of the Terra Sylvan codebase — from Firebase Auth to React Three Fiber trees to the GitHub Pages deployment pipeline. Designed so an entry-level coder can understand, navigate, and extend the project.

Next.js 16 (App Router) Firebase (Auth + Firestore) React Three Fiber TypeScript GitHub Pages (Static Export)

1 · Project Overview

Terra Sylvan is a social platform where every user profile is a 3D tree. Your tree grows as you send messages, upload media, and connect with others. The site is a fully static web app — there is no backend server. All data lives in Google Firebase.

Core concept in one sentence

User activity (messages, photos, connections) → Firestore numbers → computeTreeStats() → 3D tree geometry scales, sprouts leaves/flowers/fruit → React Three Fiber renders it.

What the site can do

  • Register / log in (email or Google)
  • Real-time 1-to-1 chat with text, images, videos, stickers, voice notes
  • Friend requests and a root-network graph
  • Daily growth ritual (journal prompt stored in Firestore)
  • Tree memory rings SVG (growth history visualisation)
  • Companion bond level (XP computed from activity)
  • Mood picker → changes the 3D sky weather
  • Forest Radio (procedural Web Audio API ambience)
  • Community forests with shared group chat
  • Stories bar
  • Admin panel (email-gated) for viewing and deleting users

8 · 3D Scene Architecture

8.1 The Lazy-Loader Pattern

React Three Fiber uses browser APIs that don't exist on the server (WebGL, AudioContext). Next.js renders components on the server by default. To prevent crashes, the 3D canvas is never imported at the top level.

// TreeScene.tsx — safe wrapper
export default function TreeScene(props) {
  const [Scene, setScene] = useState(null)

  useEffect(() => {
    // Dynamic import runs ONLY in the browser
    import('./TreeSceneCanvas').then(mod => setScene(() => mod.default))
  }, [])

  if (!Scene) return <LoadingSpinner />
  return <Scene {...props} />
}

The dashboard also wraps this in Next.js dynamic() with ssr: false as a second layer of protection.

8.2 Tree Geometry (components/3d/Tree.tsx)

Every tree type maps to a family (deciduous, conical, bamboo, palm, etc.) which picks a geometry builder function. The tree's visual scale comes from computeTreeStats().

UserProfile stats
→
computeTreeStats(profile)
→
TreeStats { scale, leafCount, flowerCount, fruitCount, rootCount, stage }
→
Tree.tsx renders geometry at that scale
// lib/tree-utils.ts — growth formula
export function computeTreeStats(profile): TreeStats {
  const ageInDays = Math.floor((Date.now() - profile.createdAt) / 86_400_000)
  const scale = 0.5 +
    (profile.messageCount    / 50)  * 0.4 +
    (profile.imageCount      / 20)  * 0.3 +
    (profile.connectionCount / 10)  * 0.3 +
    (ageInDays               / 90)  * 0.3
  // stage: seedling → sapling → young → mature → ancient
  return { ageInDays, scale, leafCount, flowerCount, fruitCount, rootCount, stage }
}

Deciduous crown profiles

  • WIDE (oak, beech, elm, cork_oak, carob, seagrape, buttonwood) — flat spreading crown, 4 lobes
  • TALL (maple, linden, teak, rubber_tree) — tall oval, lobes stacked vertically
  • FLUFFY (cherry, sea_hibiscus) — 7-puff blossom cloud, flowers forced
  • GNARLED (olive_tree, strangler_fig) — 3 irregular spheres, semi-transparent

8.3 Weather & Sky (lib/weather.ts, components/3d/DynamicSky.tsx)

The sky condition comes from two sources and is merged in TreeSceneCanvas:

  1. Real weather — useWeather() fetches from open-meteo.com using browser geolocation. Returns WeatherCondition ("clear" | "partly_cloudy" | "rain" | "storm" | "snow" | "fog").
  2. Mood override — if the user has set a mood, moodOption.weather overrides the real condition. Passed as weatherOverride prop down through TreeScene → TreeSceneCanvas → DynamicSky.
// In dashboard/page.tsx
const moodOption = mood ? getMoodOption(mood) : null
const weatherOverride: WeatherCondition | undefined = moodOption?.weather
// Passed as prop → overrides real weather in the 3D sky

How the prompt rotates

const DAY_INDEX = Math.floor(Date.now() / 86_400_000)
const prompt = PROMPTS[DAY_INDEX % PROMPTS.length]
// 30 prompts cycle, so the same prompt repeats every 30 days

Storage

On submit, saveRitual(uid, date, prompt, answer) writes to users/{uid}/rituals/{YYYY-MM-DD} and sets lastRitual on the user doc. On mount, getRitual(uid, today) checks if the doc exists — if yes, the modal doesn't auto-open.

12 · Companion Bond Level

The bond level (0-4) is computed entirely from profile stats — no separate Firestore field needed.

export function computeBondXP(profile) {
  return profile.messageCount    * 2
       + profile.imageCount       * 5
       + profile.connectionCount  * 15
       + (profile.seeds ?? 0)   * 3
       + (profile.lastRitual ? 50 : 0)

14 · Forest Radio

The Forest Radio (components/audio/ForestRadio.tsx) generates all audio procedurally in the browser using the Web Audio API. There are no MP3 files or streaming URLs.

StationHow it's generated
Rain ForestBrown noise via buffer + lowpass filter + slow LFO for thunder
Midnight Crickets5 sine oscillators at 4200-5000 Hz modulated at 18-22 Hz
Mountain WindWhite noise through bandpass filter + tremolo LFO
CampfireSparse random buffer (crackle) + sawtooth sub-bass
Tidal MangroveBrown noise + lowpass + slow frequency LFO (wave rhythm)
⚠️ Browser autoplay policy AudioContext requires a user gesture before starting. The radio only creates the AudioContext when the user clicks a station button, which satisfies this requirement.

15 · Stories

Stories work like Instagram Stories — they are visible for 24 hours. They require Cloudinary for image uploads. If Cloudinary env vars are not set, the camera/upload button is disabled.

Stories are stored in Firestore under a stories collection (not in user subcollections). The StoriesBar component queries stories from the user's accepted connections and displays them in a horizontal scroll strip below the nav.

16 · Community Forest

A Community Forest is a group space with a shared chat. Any user can create one by choosing a biome type and name. The community doc lives in the communities collection; membership in communities/{id}/members/{uid}.

The GroupChat component is a mini version of ChatWindow that writes to communities/{id}/messages. It includes a Forest Oracle feature that calls an AI endpoint (if configured) for wisdom prompts.

🚫 Common mistake to avoid If both lib/forest-toast.ts and lib/forest-toast.tsx exist, TypeScript will resolve to the .ts file first. This breaks notifications because the .ts version used require() which Turbopack doesn't support. Only forest-toast.tsx should exist.

20 · Admin Panel

The admin panel is at /admin. It checks user.email === 'roy62125@gmail.com' on every render and redirects anyone else to /dashboard. The shield icon in the dashboard nav is also conditionally shown only for that email.

⚠️ Client-side security only The email check runs in the browser. The URL itself is technically accessible to anyone who knows it. To fully protect data, your Firestore Security Rules should also restrict the getAllUsers() query to only allow reads when the requester's email matches the admin email. Use request.auth.token.email in your rules.

What the admin panel shows

  • Total member count, message count, media count, online count
  • Searchable + sortable table of all users
  • Per-user: avatar, email, tree type, growth stage, activity pills, join date, usage bars
  • Delete button → confirmation modal → removes Firestore user doc
ℹ️ Firebase Auth vs Firestore profile Deleting a user's Firestore doc removes their profile and prevents them from using the app. Their Firebase Auth account (the login itself) remains. Full deletion of Auth accounts requires the Firebase Admin SDK on a server — not possible in a static export.

22 · How to Add a New Feature

Here is the step-by-step checklist for adding any new feature to Terra Sylvan:

  1. Define the data shape — if the feature needs new Firestore fields, add them to types/index.ts first.
  2. Write Firestore functions — add getSomething() / saveSomething() to lib/firestore.ts.
  3. Write the component — add it to the right components/ subfolder. If it needs auth, call useAuth(). Mark it 'use client' at the top.
  4. Add it to a page — import and render it in the correct app/(main)/*/page.tsx.
  5. Add notifications — use forestToast.growth() / forestToast.error() etc., never bare toast().
  6. Run TypeScript check — npx tsc --noEmit. Fix any errors before pushing.
  7. Push — git add . && git commit -m "feat: ..." && git push origin main.

Example: adding a new mood option

// 1. lib/mood.ts — add to MOOD_OPTIONS array
{ type: 'foggy', emoji: '🌫️', label: 'Foggy', hint: 'Drifting through thoughts',
  weather: 'fog', accent: '#94a3b8' }

// 2. lib/mood.ts — update MoodType union
type MoodType = 'sunny' | 'breezy' | 'rainy' | 'stormy' | 'radiant' | 'foggy'

// 3. No other files need changing — MoodPicker reads MOOD_OPTIONS automatically

23 · Debugging Tips

Site not updating after push

  • Check GitHub → Actions tab for a red build. Click it and expand the failing step.
  • Common cause: TypeScript error in a new file. Run npx tsc --noEmit locally first.
  • Another common cause: export const dynamic = "force-static" missing on image-response routes.

Notifications not showing

  • Ensure only lib/forest-toast.tsx exists — if lib/forest-toast.ts also exists, TypeScript resolves to the wrong file and require() breaks under Turbopack.
  • Check browser console for "require is not defined" errors.

3D scene blank / crashes

  • Check if the import is wrapped in a dynamic import with ssr: false.
  • Three.js errors often appear in the browser console (not Next.js server logs) — open DevTools.

Firestore permission denied

  • Open Firebase Console → Firestore → Rules tab.
  • The rules must allow the operation you're trying to do. During development you can temporarily use allow read, write: if request.auth != null;.
  • For production, tighten rules to only allow users to read/write their own data.

Admin panel shows no users

  • Your Firestore rules probably don't allow reading all user docs. Add a rule: allow read: if request.auth.token.email == 'roy62125@gmail.com'; on the users collection.

Images/videos not uploading

  • Check that NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME and NEXT_PUBLIC_CLOUDINARY_UPLOAD_PRESET are set in both .env.local (local dev) and GitHub Secrets (production).
  • The upload preset must be unsigned in Cloudinary settings.
